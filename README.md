## Key Features1. Intelligent Resource AllocationThe system operates on a priority-weighted algorithm:Critical Patients: Require an ICU Bed and an ER Doctor. They are given the highest internal priority (300).Serious Patients: Require a General Bed and a Specialist (Priority 200).Normal Patients: Require a General Bed and a General Doctor (Priority 100).2. Live Dashboard & SimulationResource Panel: Features dynamic progress bars with "traffic light" color indicators (Blue for safe, Purple for moderate, Red for near capacity).Simulation Loops: The app includes a "Treatment Loop" that randomly discharges patients over time to simulate recoveries and free up resources.Auto-Allocate: When enabled, the system automatically processes the waiting queue every 1.5 seconds to fill vacant beds.3. Automated Billing SystemUpon patient discharge, the system calculates a simulated bill based on:Stay Duration: Randomly assigned length of stay.Bed/Doctor Rates: Variable costs (e.g., ICU beds at $\$1200$ vs. General beds at $\$300$).PDF Export: Automatically triggers a download of a formatted "MedAlloc AI Hospital" bill.4. Operational LoggingA dedicated Operation Log (console) provides a real-time audit trail of every action—from triage inputs and shortage alerts to resource releases—complete with timestamps.## UI/UX DesignThe interface uses a Glassmorphism design style characterized by blurred backgrounds and subtle borders.Adaptive Theme: Supports a manual and system-detected Dark Mode that adjusts all UI elements and Chart.js colors for better legibility.Responsive Layout: Uses a CSS Grid system that shifts from a three-column dashboard on desktops to a single-column stack on mobile devices.
